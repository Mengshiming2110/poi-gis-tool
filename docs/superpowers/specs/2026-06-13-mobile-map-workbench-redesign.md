# 移动端地图工作台重设计

## 背景

当前移动端按开发验证流程拆成 5 个页面：类别、区域、网格、采集、结果。这个结构便于验证核心流程，但真实用户使用时会觉得割裂：地图时隐时现，按钮像调试面板，采集成本和当前状态不够透明。

## 设计目标

- 地图始终作为主场景，用户始终知道自己在处理哪个空间区域。
- 操作放到底部任务面板里，按当前任务逐步推进。
- 减少开发术语，使用用户能理解的动作文案。
- 在采集前暴露请求预算和缓存逻辑，降低高德 API 限额风险。
- 结果页把“同步到桌面端”和“导出分享”作为主动作。

## 信息架构

移动端改为“地图工作台”：

- 顶部状态条：显示当前任务、选中类别数量、区域/网格状态。
- 地图层：常驻全屏，区域和网格直接在地图上反馈。
- 悬浮定位按钮：仅在地图步骤显示。
- 底部任务面板：承载类别、绘制、网格、采集、结果。

内部仍保留 5 个阶段，但不再表现为硬切页面，而是一个连续任务流：

1. 选择目标：选择要采集的 POI 类别。
2. 圈定区域：选择绘制工具并确认区域。
3. 采集精度：选择省额度、均衡、精细三档，生成网格。
4. 获取数据：展示请求预算、缓存命中、POI 数和停止入口。
5. 保存结果：同步云端、分享、导出、复制。

## 交互原则

- 默认可拖动地图；进入绘制模式后锁定拖动，画完自动退出。
- 每个阶段底部只有一个主按钮，主按钮文案表达下一步动作。
- 危险或消耗额度的动作前先给预算，不让用户盲点。
- 结果页主按钮是“同步到桌面端”，导出放在同一面板内但弱化。

## 视觉方向

- 使用浅色地图工作台风格，底部面板使用接近系统色的柔和背景。
- 保持 44px 以上触控目标。
- 使用分段控件、状态条和底部 sheet，减少卡片堆叠。
- 状态颜色：蓝色表示主流程，绿色表示完成，琥珀色表示额度/请求成本，红色表示错误。

## 实施范围

本次只改移动端 UI 和少量状态展示，不改高德采集核心、不改 Supabase 表结构、不改桌面端。
